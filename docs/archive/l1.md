# Web Level 1 Restructure — All Features

Apply internal layering (`api/`, `model/`, `ui/`, `lib/`, `types/`, `index.ts`) to **all** web features, extract cross-cutting auth infrastructure to `app/providers/`, and establish clear public APIs per feature.

---

## Phase A — Extract Cross-Cutting Auth Infrastructure

Move auth session/provider logic out of `features/auth/` to `app/` since every feature depends on it.

| Source | Destination |
|--------|-------------|
| `features/auth/AuthContext.tsx` | `app/providers/AuthContext.tsx` |
| `features/auth/ProtectedRoute.tsx` | `app/ProtectedRoute.tsx` |
| `features/auth/roleRedirects.ts` | `app/roleRedirects.ts` |

Update all imports across the codebase (App.tsx + every feature using `useAuth()`).

---

## Phase B — Flat Features (auth, admin, billing, reports)

### Target structure per feature

```
features/auth/
├── api/
│   └── auth.api.ts           # (was auth.service.ts — rename to .api.ts)
├── model/
│   ├── useAuth.ts            # session hook (from AuthContext / useRegistration)
│   ├── useAuthQuery.ts       # (from useRegistration.ts, renamed)
│   └── auth.types.ts         # (from register.types.ts, expanded)
├── ui/
│   ├── LoginPage.tsx
│   ├── RegisterPage.tsx
│   ├── ForgotPasswordPage.tsx
│   ├── ResetPasswordPage.tsx
│   ├── LoginForm.tsx         # extracted from LoginPage if inline
│   ├── RegisterSteps.tsx
│   ├── VerificationStep.tsx
│   ├── PersonalInfoStep.tsx
│   └── SchoolInfoStep.tsx
├── lib/
│   └── (empty for now — future utilities)
├── lazy.ts                   # re-exports from ui/ for router
└── index.ts                  # public API: export { useAuth } etc.
```

Same pattern for:
- `features/admin/` → `api/admin.api.ts`, `model/useAdmin.ts`, `ui/*.tsx`
- `features/reports/` → `api/reports.api.ts`, `model/useReports.ts`, `ui/ReportsPage.tsx`
- `features/billing/` → root-level files → `ui/`, `api/`, `model/`

### Billing special case
Billing has both root-level files AND subfolders (`budgets/`, `invoices/`).
- Move root-level pages → `billing/ui/`
- Move `billing.service.ts` → `billing/api/billing.api.ts`
- Move `useBilling.ts` → `billing/model/useBilling.ts`
- **Also apply internal structure to `budgets/` and `invoices/`:**
  ```
  billing/budgets/
  ├── api/budgets.api.ts
  ├── model/useBudgets.ts
  └── ui/*.tsx
  ```

## Phase D — Public API Barrels

Each feature (and each domain subfolder) gets an `index.ts` that exports ONLY the public API.

```ts
// features/auth/index.ts
export { useAuth, useAuthQuery } from './model';
export type { RegisterPayload, LoginPayload } from './api';
export { LoginPage, RegisterPage } from './ui'; // only if needed externally

// features/academic/attendance/index.ts
export { useAttendance } from './model';
export type { AttendanceRecord } from './api';
```

This prevents spaghetti imports and establishes clear module boundaries.

---

## Phase E — Import Cleanup

1. Update `App.tsx` to import from feature `ui/` paths or `lazy.ts`
2. Update cross-feature imports to use `index.ts` barrels
3. Delete old re-export stubs (`LazyPages.ts`) once no longer referenced
4. Verify no broken imports remain

## Implementation Order

1. **Phase A** (auth extraction) — smallest blast radius, establishes pattern
2. **Phase B** flat features — auth, admin, reports, billing root
3. **Phase B** billing subfolders — budgets/, invoices/
4. **Phase C** organized features — academic, lms, content, ai, communication, school, user, sync
5. **Phase D** barrel exports + **Phase E** import cleanup

---

## Risks & Mitigation

| Risk | Mitigation |
|------|------------|
| 300+ file moves | Do per-feature, verify with grep after each |
| `.api.ts` rename breaks many imports | Search/replace `".service"` → `".api"` in import statements |
| Cross-feature imports via old paths | After each feature move, grep for old import paths |
| LazyPages.ts re-export chain breaks | Update `lazy.ts` in each feature first, then update App.tsx |
| AuthContext move affects every component | Single global search/replace for `features/auth/AuthContext` → `app/providers/AuthContext` |
